## Methodology

Each audit follows the workflow defined in [SKILL.md](../SKILL.md):

1. **Quick Triage** -- DeFiLlama API + GoPlus token security scan
2. **Governance & Admin** -- Multisig, timelock, upgrade mechanism, token concentration
3. **Oracle & Price Feeds** -- Provider, fallback, manipulation resistance
4. **Economic Mechanism** -- Liquidation, insurance, interest/funding rate
5. **Smart Contract Security** -- Audits, bug bounty, battle testing
6. **Cross-Chain & Bridge** -- Multi-chain deployment, bridge trust model
7. **Operational Security** -- Team, incident response, dependencies
8. **On-Chain Verification** -- Verify claims against actual chain state
9. **Hack Pattern Matching** -- Cross-reference against Drift, Euler/Mango, Ronin/Harmony patterns

**Risk Rating Scale:**
- **LOW** (Triage 80-100): Strong security posture across all dimensions
- **MEDIUM** (Triage 50-79): Meaningful risks exist but mitigated by other factors
- **HIGH** (Triage 20-49): Significant unmitigated risks; exercise extreme caution
- **CRITICAL** (Triage 0-19): Immediate fund safety concerns; avoid or withdraw

**Data Sources:** [DeFiLlama API](https://api.llama.fi), [GoPlus Security API](https://api.gopluslabs.io), web research, on-chain verification.
